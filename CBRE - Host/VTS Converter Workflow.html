Outcome Automation Pipeline
1 AMF Population

Parse Exports & Apply Column Offsets

Ingest 7 xlsx files. Handle blank col A offsets (PMSetup1/2, PoolList), ScheduleListPM 3-sheet layout (Sheet3, skip rows 1-15), and WOListTR direct copy. Validate schemas and required columns.

Populate AMF Tabs

Fill 8 data tabs in processing order. Pools List runs first (builds name-to-email lookup used by all other tabs).

(OPT)Pools List PMSetupEquipSched PMScheduleList PMSetupTasksSteps WOListTR DelAngus-VTS (opt)ServiceSchedules (opt)PMGaugeReadings

Extract DelAngus-VTS Values & Auto-Map

Extract unique Angus values across 5 mapping sections (Equipment Category, Equipment Location, SR Location, SR Types, Products vs Services). Auto-map using confidence tiers from training dictionary.

Tier 1: Exact Tier 2: Fuzzy Tier 3: Manual
2 Channel Template Generation

Replicate VTS Super Converter

Transform populated AMF data into Channel Template format. Deduplicate equipment, resolve assignee emails, convert date/time formats, map recurrence periods, split SR Category+Issue, concatenate task steps, and compute all formula-dependent values directly.

Channel Template Tabs Generated

11 data tabs populated from AMF + DelAngus-VTS mappings

Teams Users Locations Equipment List PM Schedules List ServiceRequestImports TasksImports ScheduleEquipment Linking VTS Equipment Mapping UserImport Auto Assignment
3 Validation & Output

Validate Against Known-Good Outputs

Run pre-delivery checks: required fields, row counts (AMF vs export, CT vs unique IDs), assignee resolution, equipment class coverage, recurrence mapping, date formats, schedule ID cross-refs, SR category/issue coverage. Compare against completed reference files.

Split CSVs > 300 Rows

Check each CT tab for VTS importer 300-row limit. Produce split files with headers: {Tab}_part1.csv, {Tab}_part2.csv, etc. Primary candidate: ServiceRequestImports (WOListTR can have 6,000+ rows).

Generate Validation Report

Standalone file (not in the Channel Template). Lists each issue with CT tab, row, column, what's wrong, and action needed. CBRE works through flagged items in the CT before sending to VTS.

Auto-Mapping Dictionary (Phase 2 — Continuous Learning)

After each building migration is confirmed by VTS, all verified Angus-to-VTS value mappings are ingested back into the dictionary. Seeded from 3 completed AMFs (Corporate Ridge II, 777 Tower, Dexter). Each successful migration expands coverage, converting future Tier 2/3 mappings into Tier 1 exact matches. Conflicts (same Angus value mapped differently across buildings) are stored with building context and flagged for review.
